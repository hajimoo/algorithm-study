# 코딩테스트 직전 복습 가이드 (KO)

## ✅ 문제 풀이 루틴
1) 문제 전체 읽기 (입력/출력/제한 확인)
2) 입력 크기(N) 확인
3) 시간복잡도 상한 계산 (1초 기준 감각표 활용)
4) 유형 냄새 맡기: Greedy / Implementation / Brute Force / Graph
5) 정당성(왜 맞는지) or 경계(예외/범위) 체크
6) 코드 작성 + 테스트(입력 예시 검증)

## ✅ 1초 제한 감각표
- N ≤ 500         → O(N^3) 가능
- N ≤ 2,000       → O(N^2) 가능
- N ≤ 100,000     → O(N log N) 권장
- N ≤ 10,000,000  → O(N) 필요

## ✅ 구현(시뮬레이션) 루틴
상태 정의 → 명령 반복 → 다음 상태 계산 → 제약 조건 체크 → 상태 업데이트 → 출력

## ✅ 그리디 체크 질문
- 매 순간 선택지가 존재하는가?
- “지금 최선 선택”이 “전체 최선”이 되는가? (정당성 분석)
- 정렬하면 단순해지는가?
- 큰 단위가 작은 단위의 배수인가? (거스름돈류)
