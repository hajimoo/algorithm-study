# 코딩테스트 직전 복습 가이드

## 1️⃣ 문제 접근 루틴

1. 입력 크기 확인
2. 시간복잡도 상한 계산
3. 유형 판단 (그리디 / 구현 / 완전탐색 / 그래프)
4. 정당성 검토
5. 구현

---

## 2️⃣ 시간복잡도 감각표

N ≤ 500         → O(N³) 가능
N ≤ 2,000       → O(N²)
N ≤ 100,000     → O(N log N)
N ≤ 10,000,000  → O(N)

---

## 3️⃣ 구현 문제 루틴

상태 정의 → 명령 반복 → 다음 상태 계산 → 경계 체크 → 상태 업데이트

---

## 4️⃣ 그리디 체크 질문

- 매 순간 선택지가 존재하는가?
- 지금 선택이 이후 선택에 영향 주는가?
- 큰 감소가 가능한 연산이 있는가?
- 정렬하면 단순화되는가?
