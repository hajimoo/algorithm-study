# Implementation (구현/시뮬레이션) 패턴

구현 = 머릿속 알고리즘을 코드로 옮기는 과정

## 구현 문제 특징
- 알고리즘은 간단하지만 코드가 길어짐
- 문자열 처리/좌표 이동/조건 분기 많음
- 라이브러리 활용이 중요

## 풀이 루틴 (완성판)
1) 입력 받기 (격자 크기 / 상태 / 명령)
2) 상태 정의 (x,y / 방향 / 시간 / 보드 등)
3) 명령 → 변화량 매핑 (dx/dy or dict)
4) 명령 반복
5) 다음 상태 계산
6) 제약 조건 확인 (경계/장애물/충돌)
7) 상태 업데이트
8) 출력

## Tip
반복 기준이 “격자(n)”인지 “명령 리스트(plans)”인지 항상 확인
